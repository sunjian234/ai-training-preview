# AI职业技能培训与智能体应用项目官网

版本：**V0.2.0 多页面内容版**

在线预览：<https://sunjian234.github.io/ai-training-preview/>

这是一个独立、无后端、无第三方数据接口的静态项目展示网站，用于向民办职业技能培训学校校长、合作机构、企业和潜在学员介绍生成式人工智能办公、智能体应用、职业技能培训、人才培养、就业衔接和企业服务规划。

> 当前网站为项目规划展示版，不构成正式招生、收费、证书发放或就业承诺，不采集身份证、银行卡等敏感信息。

## 项目特点

- 原生 HTML5、CSS3 和 JavaScript，无框架、数据库或构建依赖。
- 采用首页加八个一级内页的多页面信息架构。
- PC 端固定主导航与下拉子目录，手机端汉堡菜单与折叠子目录。
- 支持面包屑、二级锚点导航、相关页面推荐和返回顶部。
- 所有资源使用相对路径，可直接部署到 GitHub Pages 仓库子目录。
- 不加载第三方脚本、字体、图片、统计服务或需要密钥的 API。
- 合作意向表单只在浏览器内显示反馈，不发送、不存储数据。

## 本地打开方式

推荐在项目目录启动静态服务器：

```powershell
python -m http.server 8080 --bind 127.0.0.1
```

然后访问 <http://127.0.0.1:8080/>。也可直接打开 `index.html`，但使用本地服务器更接近 GitHub Pages 的访问方式。

## GitHub Pages 部署方法

本项目无需构建。将文件提交到公开仓库后，在 **Settings → Pages** 中设置：

- Source：`Deploy from a branch`
- Branch：`main`
- Folder：`/(root)`

当前仓库已采用该方式发布。详细步骤见 [docs/GITHUB_PAGES.md](docs/GITHUB_PAGES.md)。

## 如何修改文字

1. 首页文字在根目录 `index.html` 中修改。
2. 内页文字在对应目录的 `index.html` 中修改。
3. 公共导航、页脚和交互在 `assets/js/main.js` 中维护。
4. 视觉和响应式规则在 `assets/css/style.css` 中维护。
5. 修改栏目 ID 时，同步更新该页二级锚点和公共导航中的链接。

涉及学时、收费、政策、资质、证书、就业或收益时，应继续使用“建议”“拟”“计划”“以正式信息为准”等审慎表达。

## 如何替换联系方式

当前版本没有公开真实联系人，`about/index.html` 的表单不会发送数据。增加联系方式时：

1. 只填写已获授权公开的项目电话或邮箱。
2. 不要写入私人账号、密码、Token、密钥或内部地址。
3. 如未来接入后台，必须增加隐私告知、用户授权、数据用途、保存期限、撤回与删除机制，并重新完成安全检查。

## 如何更换项目名称

应同步修改：

- 九个 HTML 页面的 `<title>`、描述和页面品牌相关文字。
- `assets/js/main.js` 中的公共页眉、页脚与版权文字。
- `README.md`、`CHANGELOG.md` 和 `docs/` 下的交付文档。
- 如仓库名变化，还需更新部署地址和全部部署说明。

不得使用未经授权的政府、协会、央企、国企、学校或合作机构名称和标识。

## 如何升级到正式网站

正式运营前至少应完成：

1. 由实际办学主体和专业人员审核招生、收费、退费、证书、就业和政策表述。
2. 明确办学主体、资质范围、师资、课程、合同、投诉渠道与责任边界。
3. 对正式联系人、合作单位、案例和标识取得可核验授权。
4. 如采集个人信息，落实最小化采集、隐私政策、授权、访问控制和删除机制。
5. 如接入报名、身份核验或支付，使用合规服务并进行独立安全评估，不在静态前端存放敏感信息。
6. 完成无障碍、兼容性、性能、链接、安全头和内容合规验收。
7. 根据实际运营主体和地区要求办理必要的备案或许可。
