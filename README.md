# AI职业技能培训与智能体应用项目官网

版本：**V0.1.0 校长预览版**

这是一个完全独立、无后端、无第三方数据接口的静态展示网站，用于向民办职业技能培训学校校长、合作机构、企业和潜在学员演示“生成式人工智能办公、智能体应用、职业技能培训、人才培养、就业衔接和企业服务”项目规划。

> 本项目仅作规划展示和合作演示，不是正式招生网站，不收款，不采集身份证、银行卡等敏感信息，不构成证书、就业、补贴或收益承诺。

## 项目特点

- 原生 HTML5、CSS3 和 JavaScript，无框架、无构建依赖。
- 单页响应式布局，适配手机、平板和电脑。
- 可直接部署到 GitHub Pages。
- 不接入数据库、统计脚本、广告脚本或需要密钥的 API。
- 合作意向表单只在浏览器内显示提示，不提交、不存储数据。
- 内容同时覆盖学校办学价值、学员成长路径和企业服务能力。
- 已加入键盘焦点、跳转链接、语义标签和减少动态效果支持。

## 本地打开方式

### 方法一：直接打开

双击 `index.html` 即可预览。网站不依赖后端，主要页面和交互可在 `file://` 模式下运行。

### 方法二：启动本地静态服务器（推荐）

在项目目录打开 PowerShell，执行：

```powershell
python -m http.server 8080 --bind 127.0.0.1
```

然后在浏览器访问：

```text
http://127.0.0.1:8080/
```

停止预览时，在终端按 `Ctrl + C`。

## GitHub Pages 部署方法

本项目无需构建。将项目文件提交到 GitHub 公开仓库后，在仓库的 **Settings → Pages** 中选择：

- Source：`Deploy from a branch`
- Branch：`main`
- Folder：`/(root)`

保存后等待 GitHub Pages 发布。完整步骤与推送前检查清单见 [docs/GITHUB_PAGES.md](docs/GITHUB_PAGES.md)。

**安全要求：任何远程仓库创建、代码推送和 Pages 启用，都必须先展示 GitHub 用户名、仓库名、公开状态、预计网址、上传文件和敏感信息检查结果，并获得用户明确确认。**

## 如何修改文字

1. 用文本编辑器打开 `index.html`。
2. 搜索要修改的中文内容。
3. 只修改标签之间的文字，尽量不要删除 `<section>`、`id`、`class` 等结构。
4. 保存后刷新浏览器检查桌面端和手机端效果。

较长文字建议保持审慎表达，涉及学时、费用、政策或资质时使用“建议”“规划”“以正式信息为准”等限定语。

## 如何替换联系方式

当前版本没有公开真实联系人，表单也不会发送数据。升级时可在“合作联系”区域增加经授权的项目联系人、电话或邮箱：

1. 在 `index.html` 中搜索 `合作联系`。
2. 在隐私说明附近添加已确认可公开的联系方式。
3. 不要把私人手机号、个人邮箱、密钥或账号凭据写入代码。
4. 若未来接入表单后台，应同步增加隐私政策、数据用途、保存期限、撤回与删除机制，并完成安全测试。

## 如何更换项目名称

在以下位置统一替换：

- `index.html` 的 `<title>`、页面头部品牌名、页脚版权文字。
- `index.html` 的 `meta description`。
- `README.md` 和 `CHANGELOG.md` 的项目标题。
- 如更换 GitHub 仓库名，同步更新部署文档中的预计网址。

不得使用未经授权的政府、协会、央企、学校或合作机构名称与标识。

## 如何升级到正式网站

正式上线前建议完成以下工作：

1. 由办学主体、法务或合规负责人审核全部招生、收费、证书、就业和政策表述。
2. 明确办学资质、招生主体、师资、课程、收费、退费、证书性质和投诉渠道。
3. 确认正式项目名称、主体信息、联系人和可公开的授权材料。
4. 如采集个人信息，增加隐私政策、用户授权、最小化采集、数据保护和删除机制。
5. 如接入支付、报名或身份核验，使用合规服务商并进行独立安全评估；不要在纯静态前端保存敏感数据。
6. 对无障碍、浏览器兼容、性能、链接、安全头和内容合规进行正式验收。
7. 按实际运营地区与主体要求完成网站备案、公安备案或其他必要手续。
