# 内容合规检查报告

检查版本：V0.2.0 多页面内容版

检查日期：2026-07-21

结论：**通过规划展示阶段自查；正式招生或运营前仍须由实际办学主体和专业人员复核。**

## 检查结果

| 检查项 | 结果 | 说明 |
|---|---|---|
| 项目性质 | 通过 | 首页醒目标注“项目规划展示版｜非正式招生网站”，公共页脚持续说明规划性质 |
| 招生与收费 | 通过 | 无正式招生时间、价格、缴费入口或收费承诺 |
| 证书性质 | 通过 | 明确学校培训合格证明不等同于国家职业资格或职业技能等级证书 |
| 国家职业方向 | 通过 | 只作衔接方向介绍，以主管部门、备案机构和正式政策为准 |
| 报考宣传 | 通过 | 明确禁止零条件报考、交费拿证、内部名额、免考取证和包通过 |
| 就业表述 | 通过 | 不保证就业，不虚构央企、国企或事业单位岗位 |
| 收益与补贴 | 通过 | 不承诺固定收入、订单、收益率或政策补贴 |
| 资质与数据 | 通过 | 未虚构合作单位、教师、资质、证书等级、人数、就业率或收入 |
| 标识与素材 | 通过 | 未使用未经授权的政府、协会、央企、学校或机构 Logo |
| Hermes 边界 | 通过 | 仅介绍公开定位与教学逻辑，不含内部代码、配置、日志或技术资料 |
| 个人信息 | 通过 | 无身份证、银行卡、账号密码或文件上传字段 |
| 第三方传输 | 通过 | 表单无 action，脚本无网络发送和浏览器持久化 |
| 付费与域名 | 通过 | 无支付、域名购买、实名认证、注册或登录功能 |
| 版本信息 | 通过 | 公共页脚显示 `V0.2.0 多页面内容版` |

## 敏感信息扫描

以下扫描结果均为 0：

- GitHub Token、API 密钥、私钥和密码赋值。
- 浏览器 Cookie、会话信息和本地绝对隐私路径。
- PDF、证书密钥文件和凭据目录。
- 其他隔离项目名称或文件。

## 高风险词语的语境复核

页面中出现的“包就业”“包拿证”“免考”“内部名额”“交费即可拿证”等词语，均位于“不承诺”“不得使用”或常见问题的否定语境中，不构成宣传承诺。

## 正式运营前待办

1. 核验实际办学、招生与证书颁发主体及其资质范围。
2. 审核招生简章、收费与退费、合同、师资、课程和证书样式。
3. 核验当期职业分类、评价机构、申报条件与政策文件。
4. 如接入表单或报名系统，建立隐私告知、授权、安全和数据生命周期机制。
5. 对正式合作单位、人员、案例、名称、Logo 和引用材料取得书面授权。
