# 手机端与响应式适配检查结果

检查版本：V0.2.0 多页面内容版

检查日期：2026-07-21

## 断点与布局策略

- `1120px` 以下：切换为汉堡菜单与侧边导航，四列内容收为两列。
- `820px` 以下：主要双栏、正文侧栏和联系区域改为单列，流程改为两列。
- `560px` 以下：卡片、按钮、表单和流程改为手机单列或紧凑布局。
- 根元素限制横向溢出；移动菜单关闭时不可造成页面水平滚动。

## 指定尺寸浏览器测试

测试使用 Edge 浏览器内核和精确 CSS 视口。

| 尺寸 | 抽测页面 | 导航模式 | 横向滚动 | 结果 |
|---|---|---|---|---|
| 390 × 844 | 关于我们 | 汉堡菜单、折叠子目录 | 无，可水平滚动距离为 0 | 通过 |
| 768 × 1024 | 学习中心 | 汉堡菜单、折叠子目录 | 无，可水平滚动距离为 0 | 通过 |
| 1366 × 768 | 学校合作 | 完整主导航、下拉子目录 | 无 | 通过 |
| 1920 × 1080 | 首页 | 完整主导航、下拉子目录 | 无 | 通过 |

## 交互与内容检查

| 项目 | 结果 | 说明 |
|---|---|---|
| 菜单打开/关闭 | 通过 | 390、768 下菜单进入视口，遮罩关闭后恢复 |
| 子目录展开 | 通过 | 展开高度正常，一级与二级层级清楚 |
| 触控目标 | 通过 | 主要按钮和导航最小高度约 44px |
| 首屏排版 | 通过 | 标题使用 `clamp()`，CTA 在小屏自然换行 |
| 卡片与流程 | 通过 | 网格随断点变为两列或单列，无固定大宽度 |
| 二级锚点 | 通过 | 横向可滚动自身目录，不扩大页面宽度 |
| 表单 | 通过 | 输入控件单列展示，反馈文字可见 |
| 页脚 | 通过 | 信息纵向排列，版本号保持可见 |
| 减少动态效果 | 通过 | 支持 `prefers-reduced-motion` |

修复记录：首次浏览器检查发现关闭状态的侧边菜单被计入文档宽度，已在根元素增加横向裁剪。复测显示 390 与 768 视口均无法产生水平滚动。

正式公开推广前仍建议使用真实 iOS 和 Android 设备复核浏览器地址栏伸缩、系统中文字体与软键盘遮挡情况。
