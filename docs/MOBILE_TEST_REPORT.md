# 手机端适配检查结果

检查版本：V0.1.0 校长预览版
检查日期：2026-07-21

## 响应式断点

- `1080px`：导航和复杂卡片开始压缩，三级课程改为单列。
- `860px`：启用移动导航；主要双栏改为单列；三列内容改为双列。
- `600px`：主要内容改为单列；按钮全宽；表单改单列；流程图与卡片适配窄屏。

## 检查结果

| 项目 | 结果 | 说明 |
|---|---|---|
| 页面宽度 | 通过 | 使用相对宽度与最大容器宽度，窄屏边距保持 12px 以上 |
| 顶部导航 | 通过 | 860px 以下使用折叠菜单，支持点击导航后自动关闭及 Esc 关闭 |
| 首屏标题 | 通过 | 使用 `clamp()` 控制字号，按钮在 600px 以下改为全宽 |
| 价值闭环图 | 通过 | 600px 以下由左右布局改为上下布局 |
| 卡片网格 | 通过 | 三列依次调整为两列、单列 |
| 三级课程 | 通过 | 平板和手机均改为单列，课程知识点在手机端保持双列 |
| 项目卡片 | 通过 | 手机端单列，字段使用短标签与自适应文本 |
| 企业流程 | 通过 | 平板三列、手机两列，移除可能重叠的连接箭头 |
| 平台流程 | 通过 | 手机端单列显示八步流程 |
| 试点班 | 通过 | 手机端主信息、人群和学习结构依次堆叠 |
| 表单 | 通过 | 手机端所有输入框单列，按钮全宽，字段具备可见焦点 |
| 页脚 | 通过 | 手机端改为纵向布局，版本号保持可见 |
| 触控反馈 | 通过 | 菜单按钮、CTA 和提交按钮具有按下状态；主要按钮高度不低于 44px |
| 横向滚动风险 | 通过 | 无固定大宽度内容；长文本可自然换行 |
| 减少动态效果 | 通过 | 支持 `prefers-reduced-motion` |

## 建议人工复核设备

正式发布前建议再使用真实设备检查：

- 360 × 800：常见小屏 Android。
- 390 × 844：常见 iPhone 逻辑尺寸。
- 768 × 1024：竖屏平板。
- 1024 × 768：横屏平板。

重点查看中文系统字体差异、浏览器地址栏伸缩、软键盘遮挡表单和触控滚动体验。
